# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 2.0.x   | ✅ Yes    |
| < 2.0   | ❌ No     |

## Reporting a Vulnerability

If you discover a security vulnerability, please:

1. **DO NOT** open a public issue
2. Contact me directly via:
   - **LinkedIn:** [https://www.linkedin.com/in/nilay-naha/](https://www.linkedin.com/in/nilay-naha/)
   - **GitHub:** [https://github.com/AmiNilay](https://github.com/AmiNilay)
3. Include details about the vulnerability
4. I will respond within 48 hours

## Security Features

- 100% client-side processing
- No data uploads
- No cookies
- No analytics
- All data stays on your device
- Optional cache with explicit user consent

## Best Practices

- Always use the latest version
- Review the code if you're concerned about privacy
- Use in incognito mode if desired
- Clear browser cache to remove all stored data

## Responsible Disclosure

I take security seriously and appreciate responsible disclosure of vulnerabilities. Please allow reasonable time for investigation and resolution before disclosing any vulnerability publicly.
